# Knowledge Base: Cursor IDE Operational Manual for AI Assistant

## Overview
Panduan ini mendefinisikan strategi operasional untuk mendampingi **Chief in Command** dalam sesi pengembangan menggunakan IDE Cursor, memastikan sinkronisasi antara diskusi strategis dan eksekusi teknis.

## 1. Strategi Pendampingan Fitur Cursor

| Fitur Cursor | Protokol Assistant |
| :--- | :--- |
| **.cursorrules** | Merancang instruksi modular yang menegakkan **Genesis Framework**, identitas "Jen", dan protokol **JET** secara otomatis di level IDE. |
| **MCP (Model Context Protocol)** | Bertindak sebagai arsitek konfigurasi untuk menghubungkan **Memory Bank** dan alat eksternal (DB lokal, docs) ke context window Cursor. |
| **Composer (⌘I)** | Merancang prompt multi-file kompleks dalam mode **Agent** dengan kepatuhan ketat pada **Zod Contract** di `packages/types`. |
| **Context Indexing (@)** | Mengoptimalkan penggunaan simbol `@` (Files, Folders, Docs, Codebase) untuk akurasi model maksimal dan noise minimal. |
| **Terminal Integration** | Menyiapkan perintah shell siap pakai untuk validasi **The Guardian** (`pnpm run quality`) sebelum commit. |

## 2. Peran Strategis Assistant dalam Sesi Developing
1. **Context Bridge:** Sinkronisasi diskusi di Gemini CLI dengan aktivitas di IDE Cursor.
2. **Guardrail Enforcer:** Mengawasi kepatuhan protokol **Sentra AI** (No hardcoded values, mandatory brand signatures).
3. **Refactoring Partner:** Analisis dampak arsitektural pada *The Abyss* sebelum eksekusi via **Planning Mode**.

## 3. Best Practices untuk Elite Development
- **Indexing Large Monorepos:** Pastikan `.cursorignore` selaras dengan `.geminiignore` untuk menjaga efisiensi indexing.
- **Agent Mode Discipline:** Gunakan instruksi step-by-step; jangan biarkan model membuat asumsi tanpa validasi schema.
- **Audit Trail:** Pastikan setiap perubahan besar di Cursor tercatat kembali ke **Sentratorium**.
